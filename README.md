# DRF_Project
### Создание платформы по онлайн-обучению с помощью Django REST framework

## Описание
В мире развивается тренд на онлайн-обучение. Но для веб-разработчика важно не только обучиться, но и знать, как реализовать платформу для онлайн-обучения. 
Поэтому новая задача касается разработки LMS-системы, в которой каждый желающий может размещать свои полезные материалы или курсы.

Ранее в проектах мы могли сразу видеть визуальное отображение результата разработки, теперь работа будет над SPA веб-приложением и результатом создания проекта будет бэкенд-сервер, который возвращает клиенту JSON-структуры.

##### Homework_30.1
+ Создано новое приложение для работы с пользователями **users**, весь его функционал доступен по адресу `/users/...` 
+ В модуле **users/models.py** создана модель пользователя `User`.
В этой модели(классе) есть поля: email, номер телефона(необязательное поле), город(необязательное поле), аватар(изображение).
Email используется как основное поле для регистрации и авторизации. 
+ Создано новое приложение для работы с материалами онлайн-платформы **materials**, весь его функционал доступен по адресу `/courses/`
+ В модуле **materials/models.py** созданы модели для курсов `Course` и для входящих в курс уроков `Lesson`.
+ В модели(классе) `Course` есть поля: название, превью(изображение) и описание. Последние два параметра не обязательны для заполнения.
+ В модели(классе) `Lesson` есть поля: название, описание, превью(изображение), видеоурок(ссылка) и курс, связанный с моделью `Course` через `ForeignKey`.
Все поля, кроме названия, не обязательны для заполнения.
+ Опишите CRUD для моделей курса и урока: 
    * для реализации CRUD для курса использован общий контроллер `CourseViewSet`, весь его функционал доступен по адресу `/courses/...`;
    * для реализации CRUD для урока использованы Generic-классы: `LessonCreateAPIView`, `LessonDestroyAPIView`, `LessonListAPIView`, `LessonRetrieveAPIView` и `LessonUpdateAPIView`.
    Все контроллеры данной модели доступны по адресу `/courses/lessons/...`.
+ Для работы контроллеров в модуле **materials/serializers.py** описаны простейшие сериализаторы `CourseSerializer` и `LessonSerializer`.

###### Шаги для локального запуска проекта
1. Клонировать проект из удаленного репозитория:
`git clone https://github.com/Sayana-All/DRF_Project.git`
2. Перейти в рабочую директорию проекта (если нужно):
`cd DRF_Project`
3. Произвести сборку и запуск контейнеров проекта:
`docker-compose up -d`

###### Настройка сервера для развертывания и удаленного запуска контейнеров проекта
1. Подключиться к виртуальному серверу Yandex Cloud:
`ssh your-user-name@publice-ip-vm`
2. Произвести обновления пакетов и библиотек на сервере:
```
sudo apt update
sudo apt upgrade
```
3. Установить на сервере Docker и Docker Compose:
```
sudo apt install -y docker.io docker-compose
sudo usermod -aG docker $USER
```
4. Настройка фаервола:
    - Проверьте состояние фаервола с помощью команды:
        `sudo ufw status `
    - Если фаервол отключен, активируйте его:
        `sudo ufw enable`
    - Откройте необходимые порты для передачи данных:
        ```
        sudo ufw allow 22/tcp   # для SSH
        sudo ufw allow 80/tcp   # для HTTP
        sudo ufw allow 443/tcp  # для HTTPS
        ```
    - После открытия портов проверьте настройки фаервола, чтобы убедиться, что правила применились:
        `sudo ufw status`
5. Настроить SSH-ключ для GitHub Actions:
    - Проверить существование ssh-ключей с помощью команды:
        `ls -la ~/.ssh/`
    - Сгенерировать ключ на сервере: 
        `ssh-keygen -t ed25519`
    - Добавить публичный ключ в `~/.ssh/authorized_keys`;
    - Проверить и посмотреть приватный ключ `id_ed25519` (для последующего сохранения в Secrets GitHub).
6. Создать рабочую директорию проекта:
`mkdir -p /drf-project && cd /drf-project`

###### Настройка Secrets в GitHub
1. Зайдите в репозиторий на GitHub и перейдите во вкладку **Settings(Настройки)** -> **Secrets and variables** -> **Actions**.
Нажмите кнопку **New repository secret** для добавления нового секрета.
2. Поочередно добавьте секреты для ключей и переменных окружения:
    * **SSH_KEY** — приватный SSH-ключ для доступа к серверу.
    * **SSH_USER** — пользователь сервера (например, test1).
    * **SERVER_IP** — IP сервера.
    * **ENV_FILE_CONTENTS** — содержимое .env-файла (скопируй из локального .env).
    * **DJANGO_SECRET_KEY** - секретный ключ Django
