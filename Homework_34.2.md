# DRF_Project
### Создание платформы по онлайн-обучению с помощью Django REST framework

## Описание
В мире развивается тренд на онлайн-обучение. Но для веб-разработчика важно не только обучиться, но и знать, как реализовать платформу для онлайн-обучения. 
Поэтому новая задача касается разработки LMS-системы, в которой каждый желающий может размещать свои полезные материалы или курсы.

## Инcтрукции
Вам предстоит «завернуть» проект, над которым вы работали на курсе DRF, в многоконтейнерное приложение и настроить на самостоятельный запуск, чтобы обеспечить быстрое масштабирование проекта при помощи контейнеризации.

## Задачи
1. Опишите файл **docker-compose** для запуска всех частей проекта единой командой.

#### Подсказки:
1) Файл **docker-compose.yaml**
    - Файл находится в корне проекта;
    - Описаны все необходимые сервисы: бэкенд, база данных (PostgreSQL), Redis, Celery, Celery Beat;
    - У каждого сервиса указаны правильные порты и зависимости между сервисами;
    - В контейнерах используются переменные окружения из файла **.env**.

2) Переменные окружения (файл **.env**)
    - Есть шаблон файла **.env**, который находится в корне проекта;
    - Все чувствительные данные вынесены в файл **.env**;
    - В файле docker-compose.yaml переменные из файла **.env** корректно подключены к сервисам через `environment` или `env_file`.

3) Инструкции в _README_
    - В _README_ четко описаны шаги по запуску проекта через docker-compose;
    - Есть команда для запуска проекта;
    - Описано, как проверить работоспособность каждого из сервисов.
